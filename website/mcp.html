Node MCP

O node MCP pode funcionar como uma tool operacional dentro do fluxo: recebe instrucao, monta argumentos, chama a API ou servidor MCP e devolve resultado estruturado para os proximos nodes.

API mode

Chamada HTTP controlada com base URL, headers, query, body e mapeamento de resposta.

External mode

Conecta a servidores MCP por Streamable HTTP, SSE ou WebSocket.

LLM mapping

Permite gerar argumentos ou normalizar output com LLM quando a integracao exigir.

MCP externo

Para integracoes como Google Workspace, AWS, Jira ou GitLab, o fluxo pode primeiro executar `listTools` e depois configurar `callTool` com argumentos seguros. OAuth e AWS SigV4 ficam no node, nao espalhados pelo prompt do agente.

{
  "mcpExternalOperation": "listTools",
  "mcpExternalTransport": "streamable_http",
  "mcpExternalAuthMode": "oauth"
}

Flow-to-MCP

Um agente Canvas Flow tambem pode ser consumido como MCP server. Cada flow salvo aparece como uma tool e clientes externos chamam `initialize`, `tools/list` e `tools/call` por JSON-RPC 2.0.

curl -X POST "https://seu-host/api/canvas-flow/mcp/default-agent" \
  -H "Authorization: Bearer <CANVAS_FLOW_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'
