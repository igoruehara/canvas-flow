Arquitetura agentica

Agent OS no Canvas Flow

O Canvas Flow usa um contrato interno para organizar papel do agente, regras, capacidades e delegação. Isso mantém a orquestração dentro do produto com agente orquestrador, skills e subagents.

Nesta página Modelo Delegação

Modelo mental

O `agentSpec` e o manifesto do agente descrevem o que o agente é, o que ele pode usar e quais limites precisa respeitar.

agents.md

Define identidade, responsabilidades e arquitetura operacional.

Guardrails e rules

Separam limites duros de comportamentos obrigatorios.

Skills e subagents

Encapsulam especialidades e contexto isolado para delegação controlada.

Delegação interna

Quando uma tarefa envolve decisão, várias tools ou integração externa, o Assistente IA pode materializar uma arquitetura com Agent Plan, agente orquestrador, skill especialista e node MCP.

Agent Plan

Registra a estratégia antes da execução, incluindo dados de entrada, validações e critério de sucesso.

Agente orquestrador

Escolhe a skill certa, respeita guardrails e decide quando chamar ferramentas.

Skill especialista

Recebe uma tarefa específica, usa os MCPs permitidos e normaliza a resposta para o orquestrador.

Edges de manifesto

Representam dependências agênticas sem transformar tudo em um caminho linear de execução.
